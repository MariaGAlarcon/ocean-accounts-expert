## Available Tone Profiles

### Academic Paper
- **Voice:** Formal, objective, third person
- **Structure:** IMRAD or topic-driven sections
- **Citations:** Extensive, APA 7th
- **Language:** Technical vocabulary appropriate to the field
- **Hedging:** Appropriate ("findings suggest" not "this proves")
- **Length:** As needed (typically 5,000-10,000 words)

### Policy Brief
- **Voice:** Authoritative but accessible, active voice
- **Structure:** Key Messages → Background → Evidence → Recommendations
- **Citations:** Minimal in-text, endnotes for details
- **Language:** Plain language, define all technical terms
- **Emphasis:** Actionable recommendations, practical implications
- **Length:** 2-4 pages maximum

### Grant Proposal
- **Voice:** Confident, persuasive, first person plural ("we propose")
- **Structure:** Per funder requirements (NIH: Significance/Innovation/Approach)
- **Citations:** Selective — cite to establish credibility and gap
- **Language:** Balance technical rigor with accessibility (reviewers may be adjacent-field)
- **Emphasis:** Innovation, feasibility, significance, track record
- **Length:** Per funder limits (strict compliance)
- **Compliance note:** Flag any AI-use disclosure requirements per the funding agency

### Email
- **Voice:** Professional, concise, direct
- **Structure:** Purpose → Context → Ask → Next steps
- **Citations:** Only if specifically relevant
- **Language:** Clear, no jargon unless writing to specialists
- **Length:** Under 300 words ideally

### Presentation / Slides
- **Voice:** Conversational, engaging, first person
- **Structure:** Story arc: Hook → Problem → Evidence → Solution → Takeaway
- **Visual emphasis:** Bullet points (max 5 per slide), key numbers highlighted
- **Language:** Simple, spoken language
- **Length:** ~1 minute per slide, 10-15 slides for 15-minute talk

### Lay Summary / Public Communication
- **Voice:** Warm, engaging, accessible
- **Structure:** Why it matters → What we found → What it means
- **Citations:** None in text, "further reading" section optional
- **Language:** No jargon, analogies encouraged, reading level ~Grade 8
- **Emphasis:** Relevance to daily life, concrete examples
- **Length:** 500-800 words

### Executive Summary
- **Voice:** Concise, authoritative, results-focused
- **Structure:** Context → Key Findings → Implications → Recommendations
- **Citations:** None or minimal
- **Language:** Business-accessible, quantify impact
- **Length:** 1 page maximum

## Process
1. Identify the target tone profile
2. Read the source content
3. Transform: structure, language, length, citation density
4. Preserve: factual accuracy, key messages, evidence basis
5. Present to human for review

## Rules
- NEVER change the substance of findings when adapting tone
- Simplifying language ≠ oversimplifying conclusions
- Flag when a claim cannot be simplified without losing important nuance
- Match the audience's knowledge level — neither condescend nor assume expertise
- If adapting for a grant: check and flag AI-use disclosure requirements
